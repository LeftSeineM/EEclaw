# EE info 运行文档

## 环境要求

- **Node.js**：18.x 或更高版本
- **npm**：9.x 或更高（随 Node.js 安装）

## 安装依赖

首次运行前，在项目根目录执行：

```bash
cd ee-info
cd /d E:\AAAA课程学习\软件设计\AAAA大模型应用\AAAA清华大学网络学堂\ee-info
npm install
```

## 开发模式运行

```bash
npm run dev
```

会同时启动：

1. **Vite 开发服务器**（默认 `http://localhost:5173`）
2. **Electron 窗口**，加载上述地址并打开开发者工具

首次启动时 Electron 会等待 Vite 就绪后再打开窗口。

## 生产构建

```bash
npm run build
```

构建产物输出到 `dist/` 目录。

## 生产模式运行

构建完成后执行：

```bash
npm start
```

将使用 `dist/` 中的静态文件启动 Electron 应用（不再依赖开发服务器）。

## 常用命令汇总

| 命令 | 说明 |
|------|------|
| `npm install` | 安装依赖 |
| `npm run dev` | 开发模式（热更新） |
| `npm run build` | 构建生产版本 |

## 登录模块

应用支持**程序化调用**清华大学统一身份认证（CAS）登录网络学堂，无需依赖浏览器窗口：

1. 点击左侧「登录」打开登录弹窗
2. 输入学号与密码，点击「登录」
3. 应用通过 HTTP 直接请求 CAS 接口，使用 SM2 加密密码后提交
4. 登录成功后，会话 Cookie 会持久化到本地，供后续爬虫等功能使用
5. **密码不会保存**，仅用于本次登录
6. 若需二次验证（短信/企业微信），程序化登录会失败，此时会回退到内置窗口方式

## 爬虫功能

登录成功后，点击顶部「刷新」可抓取网络学堂数据：

- **课程列表**：当前学期在读课程
- **公告**：各课程最新公告
- **作业**：未提交、已交未改、已批改作业

数据以 JSON 形式保存在本地 `learn-data.json`，供 UI 展示。若 Session 过期，会提示重新登录。

## 常见问题

### 端口 5173 被占用

修改 `vite.config.ts` 中的 `server.port`，或设置环境变量 `PORT`。同时需在 `electron/main.js` 中把 `http://localhost:5173` 改为对应端口。

### Electron 窗口空白

确认 Vite 已成功启动，再启动 Electron。`wait-on` 会等待 `http://localhost:5173` 可访问后再启动 Electron。

### 依赖安装失败

尝试切换 npm 镜像：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```
